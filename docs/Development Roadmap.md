# 開發優先順序建議

最後更新：2026-02

---

## 目前已完成（v0.2.1）

| 功能 | 說明 |
|------|------|
| 注音符號 On/Off | 全域切換，元件穩定 |
| 逐段朗讀 | 系統朗讀（TTS）+ 學生朗讀（STT）+ 評分回饋 |
| 中文書寫練習 | 筆順動畫 + 描紅（從學國語 port） |
| 評估報告 | 朗讀結束後的成績摘要 |
| FastAPI 後端骨架 | routes / services / models 架構完成 |

---

## 建議開發順序

### 第一優先：簡介（步驟一）

**理由：**
- 技術複雜度最低，TTS 已完成，只需新增一個介紹畫面
- 完成後六步驟有完整的「頭尾」（簡介 → 逐段朗讀 → 評估報告）
- Demo 流程更完整，方便展示給他人

**需要做的事：**
- 設計「簡介」畫面，顯示作者、背景、課文摘要（資料先用 mock）
- 加入「朗讀簡介」按鈕，用 TTS 播放內容（Google 台灣語音）
- 在 `Story` 型別加入 `intro` 欄位（作者、背景說明）
- 在 `MOCK_STORIES` 補上 intro 資料
- 新增元件 `frontend/src/components/reading-steps/Intro.tsx`

---

### 第二優先：生字練習（步驟四）

**理由：**
- 筆順動畫核心元件已完成（`frontend/src/components/stroke-order/`）
- 只需整合進學習流程，技術風險低
- 教育價值高，demo 視覺效果好

**需要做的事：**
- 逐段朗讀結束後，收集 STT 評分中錯誤率高的字
- 呈現生字清單，讓學生選擇要練習的字
- 呼叫現有的 `WriteCharacter` 元件進行筆順練習
- 練習完後造句（先用 mock，之後接 AI）

---

### 第三優先：課文理解（步驟三）

**理由：**
- 教育核心功能，蘇格拉底式 AI 問答
- 需要後端 AI API 串接（OpenAI / Gemini）
- 先完成步驟一和步驟四，demo 流程更完整後再做

**需要做的事：**
- 在 `backend/app/services/ai_service.py` 實作問答 prompt
- 前端新增問答對話介面
- 設計退出條件（答對幾題後才能繼續）

---

### 暫緩

| 功能 | 原因 |
|------|------|
| 全文朗讀（步驟五） | 邏輯和步驟二相近，等前幾步完成再做 |
| 出場卷（步驟六） | 依賴課文理解（步驟三）先完成 |
| 用戶認證（老師/學生登入） | 重要但不影響 demo，核心功能完成後再做 |
| 資料庫整合（PostgreSQL） | 目前用 mock data，等認證完成後一起做 |
