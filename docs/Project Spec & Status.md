閱讀理解平台

說明：
這個檔案記錄我目前對"閱讀理解平台"的一些想法，這檔案隨時會更新。
我過去做「學國語」時，花最多時間在 STT（語音辨識） 跟注音符號 上。因為 STT 會在朗讀扮演重要角色，我寫了一個程式來 Demo 「溫暖但堅定」的助教。請看「逐段朗讀」。

版本記錄：
0.1.0：把「學國語」的注音符號功能 port 到這平台。使用者只要點 "注音 On" 或 "注音 Off" 就可以打開或關掉注音符號。
0.2.0：把「學國語」的寫字功能 port 到這平台，並將檔案架構調整為 frontend 跟 backend。
0.2.1：把逐段朗讀從「逐句」改為「逐段」（每段包含 2–3 句），並加入「系統朗讀」功能（TTS，使用 Google 台灣語音），讓學生可以先聽範本再開始朗讀。

角色說明

老師
用「單一帳號」註冊，email確認，同意使用條款後，就可以開始使用。如果這是該校第一個老師註冊，系統會建立學校資料。
校長有特別權限嗎？例如，校長能看到全校資料嗎？這部份待未來再考慮。
老師要自己創建班級，加入學生。學生名單可以用CSV檔案匯入。
一位老師可以有多個班級，每個班級有多個學生。
加入學生時，只需要提供姓名及email。學生的email也是用「單一帳號」，要跟老師同一個學校。
老師需要上傳教材，包含課文、作者生平、課文簡介。每次上傳教材時，都要再次確認著作權。
老師可以看到每一個學生的進展。學生進展細節待定。

學生
用「單一帳號」登入後，自動找到他的老師，自動進入班級。老師如果沒有加學生，這位學生就無法登入。
這閱讀平台是設計給已經進入「讀而學」（read to learn）階段的學生，通常是三年級，但是也可能是國中生。他們可能缺乏學習興趣跟信心，需要鼓勵及引導。

平台
平台要扮演一個「溫暖但堅定」的「蘇格拉底」助教。
AI 會用在「課文理解」及「出場卷」。其他流程都在程式內完成。（嚴格說，STT/TTS 也有用到 AI，不過這兩功能已經標準化，不需要我們自己用 API 呼叫 AI。）

學生學習流程
學生登入後，會看到老師這個學期已經上傳的課文清單。
學生選課文後，進入閱讀六步驟：
1. 簡介
2. 逐段朗讀
3. 課文理解
4. 生字練習
5. 全文朗讀
6. 出場卷

簡介
系統簡單介紹作者、這篇文章、相關背景。這些資料由老師提供。
學生可以點選 Mic，讓系統用口語說明，或自己看文字。

逐段朗讀
課文被切割為數個段落（每段 2–3 句）。每一段的學習流程如下：
1. 系統朗讀：學生先按「系統朗讀」，由系統用 TTS（Google 台灣語音）播放該段文字，讓學生聽清楚正確發音。
2. 學生朗讀：學生按「開始朗讀」，系統啟動麥克風（STT），學生開始朗讀。
3. 送出評分：學生按「完成這段」，系統評分並給予回饋。
4. 進入下一段：評分通過後，自動進入下一段，重複上面流程。

因為下面因素，STT 無法完美的轉錄語音：
- 學生發音不準或漏字
- 中文裡有太多同音異義的字跟詞，容易造成錯誤
- 雜訊干擾，降低識別率

因為平台要扮演「溫暖但堅定」的助教，我做了幾個決定：
- 在逐段朗讀時，若是遇到 STT 轉錄的字同音但不同調或不同字，平台會自動更改為正確的字，減少學生的挫折感。
- 修正完後，若有 80% 的字跟原文一樣，就給學生「非常好」的鼓勵，進入下一段；若有 60% 的字一樣，就稱讚學生，進入下一段；若低於 60%，請學生再努力朗讀同一段文字。
- 注音符號可以隨時切換 On/Off，方便不同程度的學生。

請下載 https://github.com/Shinjou/lingoleap-ai-reading-tutor 到你的 PC/Mac，玩一玩，給我建議。謝謝！

課文理解
（待開發）
AI 扮演蘇格拉底式助教，透過問答引導學生理解課文大意、人物情感、段落結構。

生字練習
當學生逐段朗讀時，若有某些字經常唸錯或跳字，系統會記錄下來。在逐段朗讀結束後，系統會把記錄下來的字呈現出來，問學生是否要練習寫字。
若學生選某一個字，例如「截」，系統會先示範這個字的筆順、發音、然後請學生練習筆順。筆順練習完，系統會造二句，讓學生熟悉應用。
上面功能都是從「學國語」的 Write code port 過來。「學國語」是用 Flutter，要轉化為 TypeScript。（基本筆順動畫已完成）

全文朗讀
（待開發）
學生在完成逐段朗讀及生字練習後，進行全文連貫朗讀，評分標準較寬鬆。

出場卷
（待開發）
AI 根據課文自動出題（選擇題、填空、問答），並批改學生的回答，給出總結評語。
